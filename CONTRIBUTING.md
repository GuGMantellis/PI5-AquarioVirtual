# Como Contribuir

Obrigado por considerar contribuir para o projeto Aquário Virtual (PI5)!

## Processo de Pull Request

1. Garanta que o código siga as regras de linting e passe em todos os testes locais (veja em `tests/` e `docs/`).
2. Atualize a documentação, se necessário (ex: novas variáveis de ambiente, novas bibliotecas no `hardware/bom/ListaMateriais.csv` ou mudanças no `docs/08-documentacao/manual-usuario.md`).
3. Seu Pull Request deve usar o template fornecido em `.github/PULL_REQUEST_TEMPLATE.md`.
4. Espere pela revisão de pelo menos um membro da equipe antes do merge.

## Padrões de Commit

* Use o formato de conventional commits: `tipo(escopo): descrição concisa` (ex: `feat(firmware): adiciona leitura do sensor de pH`)
* Tipos comuns: `feat`, `fix`, `docs`, `style`, `refactor`, `test`, `chore`.
